# 📊 KIND 기업 공시 자동화 파이프라인 및 프록시 대시보드

본 프로젝트는 한국거래소(KRX) KIND '오늘의 공시' 데이터를 자동으로 수집하고, 20:00 이후에 공시된 주요 시장 경보 및 규제 리스크 정보를 5가지 카테고리로 신속히 분류하여 데이터베이스에 영구 보존(Upsert)하고 반응형 웹 포털 대시보드로 시각화하는 금융 자동화 시스템입니다.

최근 고도화를 통해 **FastAPI 백엔드 프록시 도입 및 GitHub Actions 기반 100% 클라우드 크롤링 환경**을 구축하여, 로컬 기기의 업타임 의존성 제로화와 DB 키 보안성 향상을 실현했습니다.

---

## 📂 폴더 구조 및 파일 역할

### 1. 웹 백엔드 & 프록시 서버
*   **`run_server.py`**: **FastAPI 및 Uvicorn** 엔진 기반의 대시보드 구동 웹 서버입니다.
    *   **보안 프록시**: 기존 클라이언트(HTML)에서 하드코딩으로 Supabase에 접속하던 방식을 백엔드 프록시 API(`GET /api/dates`, `GET /api/disclosures`)가 안전하게 처리하여 API Key 유출 위험을 없앴습니다.
    *   **실시간 무버퍼 캐싱**: 개발 및 모니터링 시 예전 캐시 버전을 응답하는 현상을 방지하도록 전용 `DisableCacheMiddleware`를 장착해 항상 무결성 실시간 정보를 조회합니다.
    *   **보고서 호스팅**: `reports/` 내 오프라인 백업 리포트들을 웹 상에서 마운트(`/reports`)하여 상시 조회가 가능케 구성했습니다.
*   **`vercel.json`**: Vercel Serverless Hosting을 설정하여 클릭 한 번으로 완전 무료 웹 호스팅을 연동할 수 있게 조치한 배포 프로필입니다.

### 2. 자동화 파이프라인 & 크롤링 엔진
*   **`scraper_krx.py`**: Selenium 기반 Headless 크롤러. 한국거래소 KIND에 접속하여 20:05 시점에 자동 엑셀(XLS) 다운로드를 수행합니다. GitHub Actions 서버 내 무가상화 브라우저 구동 환경에서도 에러 없이 작동하도록 최적화되어 있습니다.
*   **`read_excel_auto.py`**: 수집 엑셀 데이터 분석, 분류 및 Supabase 적재 엔진.
    *   중복 업로드를 차단하는 고유 해시 기반 `resolution=merge-duplicates` 쿼리 사양을 적용해 DB 적재를 안전하게 실행합니다.
    *   실시간으로 포트 8000번의 FastAPI 서버 실행 여부를 Ping하여 작동 중일 경우 파일 프로토콜(`file://`) 대신 웹 프로토콜 [http://localhost:8000](http://localhost:8000)으로 브라우저를 자동 연결합니다.
*   **`scheduler.py`**: (로컬 백업용) 일정을 관리하며 매일 20:05에 크롤링 파이프라인을 실행시키는 데몬입니다.

### 3. 클라우드 자동화 워크플로우
*   **`.github/workflows/kind_collector.yml`**: **GitHub Actions 스케줄러**.
    *   매일 한국 시간 저녁 **20:05(UTC 11:05)**에 알아서 깃허브 클라우드 VM 컨테이너가 켜져 크롤러 코드를 수행 후 결과를 Supabase 클라우드 DB에 적재한 뒤 자동 전원 오프를 완수합니다.
    *   사용자가 자신의 컴퓨터를 상시 켜두지 않아도 클라우드가 무제한 무료로 모든 수집 일정을 대행합니다.

### 4. 프론트엔드 및 데이터 자원
*   **`dashboard.html`**: Supabase DB와 연동되어 작동하던 통합 리스크 대시보드로, 현재는 FastAPI 백엔드 API와 실시간 동기화되어 가볍고 안전하게 렌더링을 처리합니다.
    *   **달력 네비게이션**: 상단 연/월/일 인터랙티브 캘린더 버튼을 통하여 직관적인 과거 이력 호출이 가능합니다.
    *   **다기능 필터**: 특정 날짜 내 실시간 단어 검색 외에, **'날짜x' (전역 강제 검색)** 및 긍정 검색/**'제외' (NOT 매칭)** 체크박스를 통해 필요한 정보만 추려냅니다.
    *   다크 모드/라이트 모드 자동 변환 및 "이미지 클립보드 복사(html2canvas)" 포팅을 완료했습니다.
*   **`requirements.txt`**: 파이프라인 패키지 의존성을 수록해둔 파일입니다.
*   **`test.xls` & `오늘의공시.xls`**: 모의 테스트용 공시 엑셀 데이터 템플릿입니다.

## 🏷️ 5대 카테고리 우선순위 분류 규칙

1.  **투자경고 & 투자위험 & 매매거래** (최우선 순위, `warning_risk_trading`)
    *   공시 제목 내 `투자경고`, `투자위험`, `매매거래`, `거래정지`, `정지해제`, `매매정지` 등의 키워드가 포함될 경우 최우선 배정됩니다.
2.  **단기과열** (2순위, `overheating`)
    *   공시 제목 내 `단기과열` 및 괴리율 관련 과열 지정 효력이 감지되었을 때 기재됩니다.
3.  **투자주의 & 환기종목** (3순위, `caution`)
    *   공시 제목 내 `투자주의`, `환기종목` 키워드(소수계좌 거래집중, 종가급변 등)가 들어가는 항목이 분류됩니다.
4.  **공매도** (4순위, `short_selling`)
    *   `공매도`, `공매도과열` 키워드가 들어 있는 모든 공매도 규제 공시 대상입니다.
5.  **기타** (5순위, `others`)
    *   어떤 카테고리에도 속하지 않은 일반 공시 사항이 배정됩니다.

## 🚀 로컬 사용법 및 실행 Guide

### 1. 패키지 설치 및 실행 환경 구축
로컬 가상 가동에 필요한 패키지들을 `requirements.txt`를 통해 한 번에 설치합니다.
```bash
# 가상환경 활성
source ./venv/Scripts/activate # Windows Git Bash 기준

# 필수 의존 패키지 인스톨
pip install -r requirements.txt
```

### 2. FastAPI 대시보드 웹 서버 작동 시키기
포트 8000번으로 가동하여 변경 사양 실시간 반영 및 캐시 제어 아래 안정적으로 브라우저 통신을 연결합니다.
```bash
python run_server.py
```
> **서버 주소:** [http://localhost:8000/dashboard.html](http://localhost:8000/dashboard.html) (자동 브라우저 브리지 작동)

### 3. 수동 파이프라인 강제 구동 테스트
로컬에서 임의로 파일 데이터를 새로 긁어와 웹서버가 가동되고 있는 상태에서 반영 속도를 테스트해보려면 다음 명령을 가동합니다.
```bash
python read_excel_auto.py --auto
```
*   서버 포트가 이미 상주 중이므로 감지하여 자동으로 웹 도메인 경로로 대시보드가 노출됩니다.

## ☁️ GitHub Actions 클라우드 자동 수집 가이드
1.  작업 세팅을 완료하고 본 저장소를 본인의 깃허브 원격 리포지터리로 Push합니다.
2.  해당 깃허브 저장소 페이지의 **[Actions]** 탭 메뉴판으로 진입합니다.
3.  왼쪽 탭의 `KIND Disclosure Daily Collector` 워크플로우를 선택하고 우측의 **[Run workflow]** 버튼을 통해 가상 머신 상에서 즉시 수동 수집 주기를 점검할 수 있습니다.
4.  정상 가동 시, 이후에는 **매일 저녁 20:05(KST)**마다 깃허브 클라우드가 자동 수집을 도맡으며 수집 데이터는 실시간으로 연동된 Supabase에 투명하게 동기화 저장됩니다.
