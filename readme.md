# 📊 KIND 기업 공시 자동화 파이프라인 및 리터치 대시보드

이 프로젝트는 한국거래소(KRX) KIND '오늘의 공시' 데이터를 자동으로 웹 수집(headless crawler)하고, 20:00시 이후 공시된 주요 시장 경보 및 규제 정보를 4가지 카테고리로 분류하여 인터랙티브 HTML/CSS 탭 대시보드 리포트를 자동으로 생성하는 일련의 자동화 시스템입니다.

기존의 키워드 검색 방식뿐만 아니라, 매일 밤 8시 5분에 백그라운드에서 키워드 필터링 제약 없이 전체 중요 공시를 수집 및 요약 분석할 수 있도록 개발되었습니다. 

---

## 📂 폴더 구조 및 파일 역할

### 1. 자동화 & 대시보드 엔진 (신규 자동화 파이프라인)
*   **`scraper_krx.py`**: Selenium 기반 비헤드(headless) 크롤러. 한국거래소 KIND 오늘의 공시 페이지의 엑셀 다운로드 버튼을 20:05 시점에 자동 클릭 및 타깃 다운로드합니다.
*   **`read_excel_auto.py`**: 엑셀 파싱 및 자동 탭 분류 엔진.
    *   `20:00`시 이후의 전체 공시 내역을 자동으로 취합(전체 공시 모드).
    *   수집한 공시들을 아래의 '4대 카테고리 분류 규칙'에 따라 정렬하고, gemma4 AI 종합 시장 요약을 획득합니다.
    *   최종적으로 Vanilla JS 기반 탭 토글이 탑재된 가독성 높은 반응형 HTML 대시보드 리포트(보라색/어두운 카본 테마)와 마크다운 문서를 `/reports` 내에 파일명 날짜 기반(`report_YYYYMMDD_HHMM.html/md`)으로 저장해 줍니다.
*   **`scheduler.py`**: KST(한국 표준시)를 상시 추적하며 매일 **20:05**에 `scraper_krx.py`와 `read_excel_auto.py`를 파이프라인으로 연결하여 백그라운드 실행을 제어하는 관리 스케줄러입니다.
*   **`agents_auto.md`**: gemma4 금융 분석가 AI 종합 요약 논평 출력을 위해 정의된 특수 시스템 프롬프트 명세입니다.
*   **`reports/`**: 생성 완료된 날짜별 HTML 대시보드와 마크다운 리포트가 누적 보관되는 저장소입니다.

### 2. 가상환경 및 로컬 자원
*   **`venv/`**: `pandas`, `selenium`, `lxml`, `langchain-ollama`, `beautifulsoup4` 라이브러리가 정비된 가상 구동 환경입니다.
*   **`test.xls`**: 38개의 공시 레코드(투자경고 3건, 단기과열 2건, 공매도 과열 16건 등 대용량 포함)가 포함되어 파이프라인 동작을 수동 테스트해 볼 수 있도록 영구 보존된 데이터셋 파일입니다.

### 3. 기존 소스코드 (보존)
*   **`read_excel.py` / `agents.md`**: 기존의 터미널 기반 대화형 다중 키워드 입력 필터 및 마크다운 표 단독 리포트 생성 스크립트입니다. 변경 없이 원본 유지되었습니다.

## 🏷️ 4대 카테고리 우선순위 분류 규칙

`read_excel_auto.py`를 통해 분류되는 본 대시보드는 리스크 심각도 및 사용자 요건에 따라 다음과 같은 규칙으로 4가지 카테고리로 필터링됩니다.

1.  **투자경고 & 투자위험 & 매매거래** (최우선 순위)
    *   공시 제목 내 `투자경고`, `투자위험`, `매매거래`, `거래정지`, `정지해제`, `매매정지` 등의 키워드가 포함될 경우 배정됩니다.
    *   **예외 우선순위**: 제목에 주의 문구가 섞여 있더라도(예: `[투자주의]투자경고종목 지정예고`), "투자경고" 키워드가 들어 있으므로 경고 그룹으로 우선 상향 편입됩니다.
2.  **투자주의 & 단기과열** (2순위)
    *   공시 제목 내 `투자주의`, `단기과열`, `환기종목` 키워드가 들어가는 항목들이 기재됩니다. (예: `소수계좌 거래집중`, `종가급변종목` 등)
3.  **공매도** (3순위)
    *   `공매도`, `공매도과열` 키워드가 들어 있는 모든 공매도 규제 공시 대상입니다.
4.  **기타** (4순위)
    *   그 외 어떤 카테고리 키워드에도 매칭되지 않은 당일의 주총, 공시 의무 사항 등 일반 공시가 배정됩니다.

## 🚀 사용법 및 실행 가이드

### 1. 의존성 셋업
가상환경 활성화 후 필요한 패키지 버전을 확인하거나 재설치합니다:
```bash
# 가상환경 활성화 (Windows CMD/Bash 환경 공통)
source ./venv/Scripts/activate

# 라이브러리 설치
pip install pandas lxml xlrd selenium beautifulsoup4 langchain-ollama langchain-community
```

### 2. Ollama 로컬 모델 구동
로컬 Ollama 서비스 환경이 실행 중이어야 합니다:
```bash
ollama run gemma4
```

### 3. 자동 파이프라인 상시 스케줄링 실행 (매일 20:05 실행)
스케줄러 스크립트를 서버/백그라운드 형태로 띄워두면, 매일 20:05 시점에 자동으로 크롤러를 돌려 최신 엑셀을 내려받고 분석 리포트 대시보드를 생성합니다:
```bash
python scheduler.py
```
*   만약 스케줄 일치 여부와 관계없이 **즉시 수동으로 전체 자동 크롤링 코스를 검증**해 보려면 `--test` 플래그를 추가하면 됩니다:
    ```bash
    python scheduler.py --test
    ```

### 4. 수동 실행 및 다양한 옵션 활용법
수동으로 특정 엑셀 파일을 파싱하거나, 시간 필터링 우회, AI 요약 건너뛰기 등의 옵션을 사용해 직접 대시보드 리포트를 빌드할 수 있습니다.

#### ① 일반 대화형/일반 모드 (기본값)
현재 디렉토리에서 가장 최근에 생성된 엑셀 파일을 자동으로 감지하여 **20:00 이후의 모든 공시**를 리포팅합니다.
```bash
python read_excel_auto.py
```

#### ② 자동 배치 모드 (`--auto`)
지정한 특정 파일을 분석하여 리포팅할 때 유용합니다.
```bash
# 특정 엑셀 파일(예: test.xls)을 지정하여 실행
python read_excel_auto.py --auto --file test.xls

# 자동 모드 실행 시 특정 키워드로 필터링하기 (쉼표 구분)
python read_excel_auto.py --auto --file test.xls --keywords "투자경고,단기과열"
```

#### ③ 시간 필터링 무시 모드 (`--bypass-time-filter`)
기본 적용되는 20:00시 필터링 제약을 해제하고, 해당 엑셀 파일 내에 있는 **모든 시간대의 공시 데이터**를 가공 및 분류합니다.
```bash
python read_excel_auto.py --bypass-time-filter
```

#### ④ AI 생략 정량 전용 모드 (`--no-ai`)
Ollama LLM 모델 서비스(`gemma4`) 통신을 건너뛰고, 파이썬 분석을 통한 각 탭별 통계 및 정보 메시지만을 사용하여 빠르게 HTML/MD 리포트를 출력합니다.
```bash
python read_excel_auto.py --no-ai
```

## 💻 리포트 확인 방법
* 모든 실행 성공 시 `reports/` 폴더 하위에 날짜 기반 파일명(`report_YYYYMMDD_HHMM.html`, `.md`)으로 생성됩니다.
* **대시보드 보기**: 생성된 `reports/*.html` 파일을 크롬/엣지 등의 웹 브라우저로 열면 어두운 테마와 반응형 탭이 완벽히 작동하는 최첨단 리스크 리브리핑 대시보드를 직접 사용하실 수 있습니다.
